This guide covers upgrading from v0.23.x to v0.24.0. This is not a drop-in replacement — you will need to update configuration files, choose an authentication mode, and run a data migration before the application will start.

See the [release announcement](/posts/2026-02-18-release-notes-v0.24.0) for an overview of what's new.

## Before You Start

1. **Back up everything.** Redis data, configuration files, environment variables.
2. **Decide on an authentication mode.** This determines your system requirements (see below).
3. **Review your system resources.** Full mode requires PostgreSQL 17+ and RabbitMQ 4.3+ to be installed and accessible.

## Step 1: Choose an Authentication Mode

v0.24.0 introduces two operational modes. This choice affects your infrastructure requirements and must be made before proceeding.

**Simple mode** — The default. Uses Redis-backed authentication. No additional infrastructure. Suitable for most self-hosted deployments where a single operator manages access.

**Full mode** — Adds Rodauth-based authentication (MFA, WebAuthn, SSO), organizations, billing, and background job processing. Account data, sessions, and operational state are stored in PostgreSQL. Background jobs (email delivery, notifications, webhook processing) are handled by RabbitMQ.

**Disabled** — No authentication. The application operates in anonymous-only mode. Useful behind a VPN or reverse proxy that handles auth externally.

## Step 2: Update Configuration Files

Several configuration keys have been renamed, moved, or added. The application will not start with outdated configuration.

| Change | Before | After |
|--------|--------|-------|
| Email sender name | `fromname` | `from_name` |
| Auth environment variables | various names | `AUTH_SIGNIN_ENABLED`, `AUTH_SIGNUP_ENABLED` |
| Session configuration | `auth.yaml` | site config |
| Domains configuration | `site` section | `features` section |
| Regions configuration | `site` section | `features` section |
| Billing configuration | nested under other keys | top-level `billing` section |

### New Configuration Keys

| Key | Description | Default |
|-----|-------------|---------|
| `site.secret_options.generated_value_display_ttl` | Seconds a generated password remains visible on the receipt page. Set to 0 to disable. | `60` |
| `allowed_signup_domains` | Restrict signups to specific email domains. | _(none)_ |
| `logging.yaml` | HTTP request logging configuration file. | _(new file)_ |
